# 设计系统 · Design System

## 色彩体系

仪表盘整个 UI 以 **Tailwind Stone** 色系为基底，营造温暖、沉稳的阅读氛围。图表配色在此基础上降饱和、加热调，使数据可视化与页面整体风格统一。

### 页面基调

| 角色 | 色值 | 说明 |
|------|------|------|
| 背景 | `#f5f0ed` | 暖米白，stone-100 方向 |
| 卡片 | `#ffffff` + 柔和弥散阴影 | 无边框，通过阴影层次表达纵深 |
| 正文 | `#1c1917` / `#2d2927` | 近黑但保留暖意 |
| KPI 主数字 | `#f5b642` | 明亮暖金，替代原 amber-600，用于消费总额/游戏总数/剧集总数 |
| KPI 次级数字 | `#6b7280` | 冷灰，减少视觉干扰（如硬件投入、游戏消费等辅助指标） |
| 强调/按钮 | `#d97706` (amber-600) | 按钮、链接等交互元素 |
| 边框/分隔 | `#e5e0dc` | stone-200 |

### 设计原则

**"加白提亮、微调色相、干净素雅"**。平台品牌色在仪表盘中不是品牌标识，而是数据系列编码。它们需要在不争夺注意力的情况下清晰区分。因此：

1. **加白提亮** — 从品牌原色出发，混入白色来提高明度，取代旧的"加灰降饱和"方式，避免颜色浑浊灰暗
2. **微调色相（Hue Shift）** — 将过于刺眼的原色向更优雅的方向偏移（如红→珊瑚，绿→鼠尾，蓝→天蓝）
3. **饱和度自然下降** — 因混入白色而比例性降低饱和度，而非人为加灰，保持颜色干净清爽
4. **明度统一** — 所有色块在白色卡片上保持 L ≈ 65-80 的明度区间，视觉重量均匀

### 各图表配色决策

**环形图（金额/时间/类型分布）**
- 直接使用平台色作为扇区填充色
- 按数值降序排列，从 12 点钟方向顺时针展开
- 悬停偏移 `hoverOffset: 4`

**月度趋势图（柱状+折线混合）**
- 堆叠柱状图：平台色（与环形图一致）
- 游戏时长折线：`#d8c898`（香槟金），无数据点标记
- 剧集时长折线：`#d8a8b8`（灰玫），无数据点标记
- 合计时长填充区：`rgba(216, 200, 152, 0.12)`，半透明香槟金
- 三条线均 `pointRadius: 0`，`pointHoverRadius: 4`，保持平滑曲线的同时也保留 tooltip 触发

**价格-时间散点图**
- 各平台数据点使用对应平台色 + 70% 不透明度（`color + 'B3'`）
- 数据点添加 1px 白色描边（`borderColor: '#ffffff'`），使圆形轮廓清晰锐利
- 右下角冷青薄雾渐变 `rgba(106, 184, 238, 0)` → `rgba(106, 184, 238, 0.10)`，暗示"右下 = 性价比最优"方向
- 评分通过点半径编码：未评分 2px → 评分 10 为 4px
- 高分游戏（≥8 分）叠加光环效果：半径 +3px、~19% 不透明度的同色光晕

**游戏分类环形图**
- 使用 HSL 动态生成色相，固定饱和度 30%、明度 55%
- `hsl(${hashCode(label) % 360}, 30%, 55%)`，确保所有分类颜色在同一饱和度水平

### Tooltip 样式

- 白色背景 + 柔和弥散阴影（`0 10px 30px rgba(0,0,0,0.06)`），与卡片风格一致
- 使用 `position: fixed` 渲染在 `document.body`，避免被父容器遮挡
- 月报 tooltip 内嵌 Top 5 收支和游玩排行

### KPI 排版层级

- **标题**（如"消费总览"）：`font-normal`（400 字重）+ `text-stone-500`，轻盈、低干扰
- **主数字**（消费总额、游戏总数、剧集总数）：`font-bold` + Georgia 半衬线字体 + `#f5b642` 暖金，沉稳突出
- **次级数字**（硬件投入、游戏消费等）：`font-semibold` + `#6b7280` 冷灰，信息完整但不抢眼
- **¥ 符号 / 单位**（"款""小时"）：比数字小 15%（`0.85em`），降低一个色度（`opacity: 0.65`）
