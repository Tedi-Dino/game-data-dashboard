# 设计系统 · Design System

## 色彩体系

仪表盘整个 UI 以 **Tailwind Stone** 色系为基底，营造温暖、沉稳的阅读氛围。图表配色在此基础上降饱和、加热调，使数据可视化与页面整体风格统一。

### 页面基调

| 角色 | 色值 | 说明 |
|------|------|------|
| 背景 | `#f5f0ed` | 暖米白，stone-100 方向 |
| 卡片 | `#ffffff` + `#e5e0dc` 边框 | 洁净但不过于冷峻 |
| 正文 | `#1c1917` / `#2d2927` | 近黑但保留暖意 |
| 强调 | `#d97706` (amber-600) | 按钮、KPI 数字、链接 |
| 边框/分隔 | `#e5e0dc` | stone-200 |

### 设计原则

**"低饱和、暖色调、可辨识"**。平台品牌色（如任天堂红、Steam 蓝、Xbox 绿）在仪表盘中不是品牌标识，而是数据系列编码。它们需要在不争夺注意力的情况下清晰区分。因此：

1. **饱和度降低 30-40%** — 从品牌原色退到柔和色域，不刺眼
2. **冷色加暖** — 纯蓝/纯绿向灰/褐方向偏移，融入 stone 环境
3. **明度均衡** — 所有色块在白色卡片上保持相近的视觉重量
4. **特殊色修复** — Epic `#f2f2f2` 在白色背景上近乎不可见，改为可辨识的暖灰

### 各图表配色决策

**环形图（金额/时间/类型分布）**
- 直接使用平台色作为扇区填充色
- 按数值降序排列，从 12 点钟方向顺时针展开
- 悬停偏移 `hoverOffset: 4`

**月度趋势图（柱状+折线混合）**
- 堆叠柱状图：平台色（与环形图一致）
- 游戏时长折线：`#c4a040`（暗金），无数据点标记
- 剧集时长折线：`#c4556a`（灰玫红），无数据点标记
- 合计时长填充区：`rgba(196, 160, 64, 0.12)`，半透明暖金
- 三条线均 `pointRadius: 0`，`pointHoverRadius: 4`，保持平滑曲线的同时也保留 tooltip 触发

**价格-时间散点图**
- 各平台数据点使用对应平台色 + 70% 不透明度（`color + 'B3'`）
- 右下角暖琥珀渐变 `rgba(212, 160, 23, 0)` → `rgba(212, 160, 23, 0.18)`，暗示"右下 = 性价比最优"方向
- 评分通过点半径编码：未评分 2px → 评分 10 为 4px

**游戏分类环形图**
- 使用 HSL 动态生成色相，固定饱和度 30%、明度 55%
- `hsl(${hashCode(label) % 360}, 30%, 55%)`，确保所有分类颜色在同一饱和度水平

### Tooltip 样式

- 白色背景 + stone-200 边框 + 圆角阴影，与卡片风格一致
- 使用 `position: fixed` 渲染在 `document.body`，避免被父容器遮挡
- 月报 tooltip 内嵌 Top 5 收支和游玩排行

### 数据表格状态色

| 状态 | 左边框色 | 说明 |
|------|---------|------|
| 进行中 | `#10b981` (emerald) | 活跃进行标识 |
| 待玩 | `#f59e0b` (amber) | 待处理提醒 |
| 已通关 | `#3b82f6` (blue) | 完成标识 |
| 已放弃 | `#f43f5e` (rose) | 终止标识 |
| 全成就 | `#d4a017` (gold) | 特殊荣誉行，覆盖其他状态色，带脉冲动画 |
